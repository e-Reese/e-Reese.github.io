Building a Travel Router with Raspberry Pi

By Ethan Reese August 7, 2025

My complete travel router setup using a Raspberry Pi 4

My purpose for building this travel router was to solve the problem of hotel Wi-Fi that limits connections to one device and to create a secure network while traveling. In this tutorial, I'll walk you through how I built a portable travel router using a Raspberry Pi that solved these problems.

What You'll Need

  • A laptop or desktop computer
  • Raspberry Pi (I used a Pi 4, but Pi 3B+ works too)
  • MicroSD card (16GB or larger recommended)
  • USB Wi-Fi adapter (for dual Wi-Fi capability)
  • USB-C 5v Power Supply (for powering the Pi)
  • A monitor, keyboard, and mouse (my prefered method)
  • Mini screen and keyboard (for passing guest portal login pages)
  • Ethernet cable (optional for connecting via ssh)
  • Portable power bank (optional)

Raspberry Pi OS Setup

Step 1: Download Raspberry Pi Imager

First, download the Raspberry Pi Imager from the official Raspberry Pi site.

Once downloaded, open the imager and select the Raspberry Pi OS (64-bit) image. Select the SD card and click "Next", then "Write" to flash the image to the SD card.

Raspberry Pi Imager Settings

After flashing, insert the SD card into your Raspberry Pi, connect it to your monitor, keyboard, and mouse, and power it on. Alternatively, you can connect it to your computer via Ethernet and use the terminal to configure it (official documentation here).

    Wi-Fi Antenna Configuration

    Now we'll set up the dual Wi-Fi configuration - one antenna to connect to an existing network, and one to create our own network.

    Step 3: Install WiFi Antenna drivers

    This process is slightly different depending on the type of antenna you're using. I chose the RTL8811CU found here. The RTL8811CU was not the best choice, as it wasn not plug and play, however there are many other recommended antennas that can be found online. Below is the process for the RTL8811CU: 

    # Install required headers
sudo apt install -y raspberrypi-kernel-headers build-essential bc dkms git

# Create a directory for the driver
mkdir -p ~/src

# Access the directory
cd ~/src

# Download driver repository.
git clone https://github.com/morrownr/8821cu-20210916.git

# Access the driver repository.
cd ~/src/8821cu-20210916

# Install the driver.
sudo ./install-driver.sh

# Reboot
sudo reboot

    If you're using a different antenna, the process will be different. For the documentation I used to get the RTL8811CU working, see here. I found that I needed to skip the "arm_64bit=0" step.

    Basic Configuration

    Now that RaspAP is running and the second Wi-Fi antenna drivers are installed, you'll need to access RaspAP's web interface to configure it.

    Step 4: Configure RaspAP

    Navigate to 10.9.141.1 in your web browser on your Pi. The default username is "admin" and the default password is "secret".

    It is likely that the builtin antenna is faster and has better range than the external antenna. This means that we need to switch which antenna is used for a hotspot and which is used to connect to the internet.

    Go to the "Hotspot" tab on the left side of the screen and set the "Interface" to the external antenna (wlan1 likely).

    Go to the "Security" tab at the top and change the password for the hotspot.

    Go to the "Advanced" tab at the top and confirm the country code is set to the country where you're located.

    Now that the settings are configured in the RaspAP interface, we'll need to clean up old WiFi configurations still in the .conf files.
    In terminal: 

    # Delete old Wlan0 configuration
sudo nano /etc/dhcpcd.conf

    Delete the five lines for wlan0, starting with the line that says "# RaspAP wlan0 configuration". Use arrow keys to navigate to the line and Control + K to delete the line.
    The "RaspAP wlan1 configuration" lines should be left alone.
    Save and exit the file by pressing Control + X, then Y, then Enter.

    Next, delete any pre-configured network entries in the system.
    In terminal: 

    # Delete pre-configured network entries
sudo rm /etc/NetworkManager/system-connections/preconfigured.nmconnection

# Then, delete any WPA Supplicant entries by removing the wpa_supplicant.conf file.
sudo rm /etc/wpa_supplicant/wpa_supplicant.conf

# Finally, reboot the system.
sudo reboot

    Step 4: Configure Wi-Fi Interfaces

    In the RaspAP interface, navigate to Network → Wireless and configure both interfaces:

    • The built-in Wi-Fi: Set as an access point (creates your network)
    • USB Wi-Fi adapter: Set as a client (connects to hotel/external Wi-Fi)
    # Example configuration in /etc/config/wireless
config wifi-device 'radio0'
    option type 'mac80211'
    option channel '36'
    option hwmode '11a'
    option path 'platform/soc/fe300000.mmcnr/mmc_host/mmc1/mmc1:0001/mmc1:0001:1'
    option htmode 'VHT80'
    option country 'US'

config wifi-iface 'default_radio0'
    option device 'radio0'
    option network 'lan'
    option mode 'ap'
    option ssid 'TravelRouter'
    option encryption 'psk2'
    option key 'yourpassword'
